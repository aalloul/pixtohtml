PHP Object caching

Stores/Restores member(s) of a class to/from static files.

Features:

  • Use with any object. The Cacheable provides static methods that can be used to cache any object
  • Use with objects that extend Cacheable. The caching can now be done partially or completely internaly if your class extends Cacheable!
  • Cache expire. On cache store you can define expiration rules. If you donâ€™t - the cache never expires!
    • Expire after some time: the cache will expire after the given ammount of seconds pass since it was created.
    • Expire when another file is changed: the cache will expire when the modify time of a file changed.
  • Custom expiration rules. You can implement custom expration rules by extending the CacheExpire class
  • Object state. You can have multiple caches for one object. Provide the object state to specify which cache you want to use.
  • Fast restore. The restore function is cheap. The cache itself is php. It takes one include, some checks to see if the included cache is valid and one assign per stored member.

PHP Class Install (and Update) system

Uses the PhpDoc @version and @history to execute migrations b/n versions @version must be in valid php version format @history must be the name of the class that extends InstallHistory and defines the changed for each version. Optionally a relative path to the file that holds the history class can be supplied. Example:

/**
 * @history MyHistory path/to/MyHistory.php
 * @version 1.0.1-alpha5
 */
class MyModule{
    ...
}

Features:

  • Perform update on multiple classes with single function call

    Install::create(array('MyModule'))->sync();
  • Both upgrade and downgrade are supported
